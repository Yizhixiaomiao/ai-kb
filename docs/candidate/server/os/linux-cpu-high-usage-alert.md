## 适用范围

- 适用于服务器、业务机器、Node 节点出现 CPU 使用率达到或超过 90%/95% 的告警。
- 适用于规则名称包含“CPU使用率达到95%”“CPU使用率超过95%”“业务机器CPU使用率”等场景。

## 常见现象

- CPU 使用率持续高于告警阈值。
- 业务响应变慢、接口超时或应用日志出现大量慢请求。
- 某个进程、容器或 Pod 长时间占用 CPU。

## 判断依据

- 先确认是单台主机、容器、Pod 还是整个业务集群 CPU 升高。
- 区分短时尖峰和持续高负载。
- 如果告警来自 Kubernetes 节点或 Pod，应优先结合 Kubernetes 运维手册查看资源对象。

## 处理步骤

1. 确认告警主机、业务系统、发生时间、持续时长和影响范围。
2. 查看 CPU 总体占用、负载、进程排名和最近变更记录。
3. 定位高 CPU 来源：应用进程、数据库进程、容器、批处理任务、备份任务或异常循环。
4. 对应用进程优先查看应用日志、接口流量、线程状态和发布变更。
5. 对容器或 Kubernetes 场景，查看容器资源限制、Pod 状态、事件和重启次数。
6. 如果业务受影响，按预案执行扩容、限流、重启实例、回滚发布或转交对应系统负责人。
7. 记录高 CPU 进程、处理动作、业务影响、恢复时间和是否需要优化阈值。

## 验证方式

- CPU 使用率回落到告警阈值以下并保持稳定。
- 业务接口、应用日志和监控指标恢复正常。
- 告警恢复或不再持续触发。

## 注意事项

- 不要直接 kill 进程或重启服务，必须先确认业务影响和回退方案。
- 涉及生产业务扩缩容、重启、回滚时应按变更流程执行。
- 如果 CPU 告警频繁但业务无异常，应复核告警阈值、采集周期和排除短时尖峰。
