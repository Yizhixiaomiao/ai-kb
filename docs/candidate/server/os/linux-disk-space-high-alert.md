## 适用范围

- 适用于服务器磁盘空间使用率超过 90%/95%/97%、磁盘剩余空间小于阈值的告警。
- 适用于规则名称包含“磁盘空间使用率”“磁盘空间剩余小于”等场景。

## 常见现象

- 文件系统使用率过高。
- 日志、备份、临时文件、容器镜像或数据库文件占用空间。
- 应用写入失败、数据库异常或系统服务异常。

## 判断依据

- 区分磁盘容量不足、inode 耗尽和挂载点异常。
- 先确认告警挂载点，不要只看根目录。

## 处理步骤

1. 确认告警主机、挂载点、阈值、使用率和业务影响。
2. 查看文件系统容量、inode、目录占用和近期增长最快的目录。
3. 判断占用来源：应用日志、数据库文件、备份文件、容器镜像、临时文件或异常 core dump。
4. 清理前确认文件归属、保留策略、备份要求和是否允许删除。
5. 对日志类文件优先采用归档、压缩、日志轮转或调整保留策略。
6. 对数据库、容器和业务目录，必须联系对应负责人确认后处理。
7. 必要时扩容磁盘或调整挂载，记录处理动作和容量变化。

## 验证方式

- 告警挂载点使用率低于阈值。
- inode 使用率正常。
- 应用写入、数据库状态和业务访问正常。

## 注意事项

- 不要删除不明文件、数据库文件、正在写入的日志或备份链。
- 清理生产文件前必须确认业务归属和回退方案。
- 频繁磁盘告警应推动日志轮转、备份保留和容量规划优化。
