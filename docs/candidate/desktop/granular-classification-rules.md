# 桌面工单细分类规则

本规则用于后续从工单生成工程师参考知识，避免过度合并。

## 基本原则

- 一个文档只覆盖一个明确处理入口。
- 如果故障现象相同但处理路径不同，应拆成不同文档。
- 如果涉及不同系统或不同责任团队，应拆成不同文档。
- 大类文档只做导航，不作为主要推荐结果。

## 打印机拆分规则

不要把所有打印问题合并为“打印机无法打印”。至少按以下场景拆分：

- 共享打印机连接失败。
- 驱动安装失败或驱动丢失。
- 打印机脱机、暂停、队列卡住。
- 卡纸、进纸失败、出纸异常。
- 标签打印机纸张识别、库位码、标识卡。
- 扫描后无法预览或下载。
- 打印端口被安全软件删除。
- 打印模板、纸张规格、业务单据格式异常。
- 打印服务器或打印服务异常。

## 设计软件拆分规则

CAD / SolidWorks / PLM 相关问题不应合并为“软件安装”。

建议拆分：

- CAD 安装、卸载、重装失败。
- CAD 打开后卡死、崩溃、无响应。
- CAD 图纸打不开、文件损坏、加密相关异常。
- CAD 与 PLM 接口异常。
- SolidWorks 许可、激活、证书异常。
- SolidWorks 安装源缺失或无法重装。
- SolidWorks 界面布局、菜单栏、功能区异常。
- 设计软件性能问题。

## 业务系统拆分规则

业务系统应按系统名和问题入口拆分：

- OA：登录、附件上传、打印、浏览器兼容。
- PLM：客户端、登录、接口、传图。
- MES：登录、车间终端网络、页面访问。
- WMS：库位码、标签打印、客户端组件。
- 云盘：安装、同步、共享权限、文件恢复。

## 硬件拆分规则

不要把硬件问题都合并为“无法开机”。

建议拆分：

- 不通电。
- 主机通电但无显示。
- Windows 蓝屏。
- 黑屏但鼠标可见。
- 内存故障或内存不足。
- 硬盘故障或读取困难。
- C 盘空间不足。
- 显示器、键鼠、外设故障。

## 网络与安全客户端拆分规则

建议拆分：

- 网络认证失败。
- 认证设备数量超限。
- IP 冲突或 DHCP 异常。
- VPN 或远程访问失败。
- 加密客户端掉线。
- DLP / 桌管策略拦截。
- 浏览器证书或控件异常。

## 推荐排序建议

工单系统推荐时应优先推荐细分类文档。如果只命中大类文档，应提示“需要进一步选择场景”。

示例：

```text
用户输入：打印机无法打印

先追问或自动判断：
- 是否连接不上？
- 是否脱机？
- 是否卡纸？
- 是否标签纸识别失败？
- 是否业务系统打印格式异常？
- 是否驱动安装失败？
```
